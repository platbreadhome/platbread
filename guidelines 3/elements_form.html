디자인 원칙

설명

본 가이드라인은 플랫브레드 플랫폼 서비스 디자인 설계를 다룹니다.
표준화된 색상, 레이아웃, 타이포그래피 및 컴포넌트 스타일의 정의와 사용 규칙을 제공하여 브랜드를 일관성을 유지하고 사용자에게 명확하고 안정된 시각 경험을 제공합니다. 이를 통해 브랜드 인식과 효율적인 커뮤니케이션과 일관된 의사결정을 지원합니다.

타이포그래피

실제 웹페이지에서 렌더링시 폰트 줄간격과 피그마에서의 줄간격이 달라 버튼 텍스트에 대해 line-height (140%) 중심으로 설정변경하여 디자인과 웹페이지에서의 동일한 수치가 나올수있도록 한다

서체 : Noto Sans KR
기본 사이즈 : 13px

색상 팔레트

설명

브랜드 경험에서 색상은 단순한 시각 요소가 아니라 브랜드의 상징이며, 스타일과 의미를 전달하는 핵심도구입니다.

색상 팔레트 구조 : 모든 색상은 HSL 기준으로 최대 13단계 명도로 구성해 다양한 환경에서도 일관성과 균형을 유지합니다.
적용 원칙 : 디자이너는 브랜드 컬러를 최소 11단계로 확장·활용하고 변경 시 반드시 공유하여 팀의 통일성을 보장합니다.

UI 공통 색상 단계 가이드
텍스트 · 배경 · 보더 색상은 컴포넌트 UI의 성격에 따라 단계값을 달리 적용할 수 있습니다.
단, 프라이머리·세컨더리·터셔리·강조·시스템 색상은 정해진 단계 체계에 맞춰 일관되게 사용해야 합니다.
